# XCN Similarity 요구사항 정리

## 목적

인터넷 구간 로깅 장비가 수집/복호화하여 저장한 텍스트 및 첨부 데이터와 사내 보유 문서를 임베딩 벡터로 비교해 내부정보 유출 가능 경로를 추적한다.

## 주요 기능

1. 사내 보유 문서를 등록하고 임베딩 모델로 벡터화하여 저장한다.
2. 로깅된 본문/첨부 텍스트를 벡터화하고 등록 문서와 유사한지 검색한다.
3. 등록 문서를 기준으로 과거/현재 로깅 데이터에서 유사한 데이터를 역검색한다.
4. 사용자가 문서를 등록, 삭제, 재처리할 수 있는 관리 기능을 제공한다.
5. 검색 결과는 단순 점수가 아니라 매칭된 문서/로그/청크 근거를 제공한다.

## 현재 내부 데이터 확인 결과

상세 원천 데이터 조회 규칙은 [EMS_DATA_SOURCE.md](EMS_DATA_SOURCE.md)를 기준으로 한다.

MongoDB:

```text
mongodb://10.10.20.6:27018/venus
collection: EMS_MESSAGE_202605
```

확인 기준:

```text
svc prefix가 X 또는 U인 데이터 제외
```

내부 데이터 산정:

```text
컬렉션 범위: 2026-05-01 00:00:01 ~ 2026-05-27 10:50:37
전체 건수: 약 7,681,739건
X/U prefix 제외 대상: 272,100건
X/U prefix 제외된 건수: 7,409,639건
대상 일평균: 약 1만건/day
첨부 포함 샘플 비율: 약 25.3%
본문 size 샘플 평균: 약 3.3KB
첨부 size 샘플 평균: 약 2.5KB
```

고객사 설계 기준은 내부 데이터의 10배로 잡는다.

```text
로그 벡터화 대상: 약 10만건/day
첨부 포함: 약 2.5만건/day
월간 대상: 약 270만건
1년 대상: 약 3,600만~3,700만건
보수적 청크 기준: 최대 1억 vectors/year
```

## 기존 시스템 전제

```text
메타데이터 저장소: MongoDB
첨부파일 저장소: MinIO
복호화 본문: 텍스트 저장
보관 기간: 1년
트래픽 분류: 첨부/웹메일/메신저/게시판 업로드 구분 가능
망 환경: 외부망 차단
GPU: NVIDIA L4 24GB 1장 기준
OCR: 1차 제외
```

## 추가 확정 필요 사항

1. 고객사 실제 본문 평균/중앙값/P95/P99 크기
2. 고객사 첨부파일 유형별 건수와 P95/P99 크기
3. 1차 지원 파일 포맷 범위
4. 준실시간 허용 지연 시간
5. 문서 보안등급/부서별 접근 통제 정책
6. 삭제 시 원본/추출 텍스트/벡터의 물리 삭제 필요 여부
7. 오탐/정탐 피드백을 모델 튜닝에 활용할지 여부
